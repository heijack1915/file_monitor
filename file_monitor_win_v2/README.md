# Windows 文件监控工具 v2

一个跨平台文件监控工具，提供 Web 界面，可在 Windows 10/11 上实时监控系统文件变化。

## 功能特性

- **实时监控**：基于 watchdog（Windows ReadDirectoryChangesW 后端）
- **进程识别**：使用 psutil 识别触发文件操作的进程
- **Web 界面**：深色主题，支持实时事件推送（SSE）
- **AI 分析**：支持 OpenAI 兼容接口进行智能分析
- **离线规则**：内置规则引擎分析进程和路径
- **低资源占用**：设计为最小系统开销


## 进程识别策略

1. **目录缓存**：5 秒 TTL，相同目录事件直接命中
2. **psutil 查询**：open_files → cwd → cmdline 依次匹配
3. **路径规则**：路径特征推断进程名称

准确率：约 85%（取决于操作速度）

## 系统要求

- Windows 10/11
- Python 3.8+
- 无需管理员权限（用户级运行）

## 安装运行

### 方式一：直接运行（推荐）

1. 下载或克隆本项目
2. 双击 `run.bat` 或在命令行运行：
   ```cmd
   python app.py
   ```
3. 打开浏览器访问：http://localhost:5006

### 方式二：打包运行

```cmd
build.bat
```

输出：`dist/FileMonitorV2/FileMonitorV2.exe`

## 使用说明

1. 打开浏览器访问 http://localhost:5006
2. 选择要监控的路径（桌面、文档、下载等）
3. 点击「开始监控」
4. 查看实时文件事件
5. 点击「AI 分析」或「离线分析」获取事件详情

## 常见问题

### Q：进程显示为 "unknown"
- 文件操作速度太快，psutil 无法捕获
- 建议手动操作（复制/粘贴文件）进行测试

### Q：CPU 占用高
- 减少监控路径数量
- 避免监控包含大量文件的目录

### Q：事件不显示
- 检查路径是否存在
- 尝试手动添加自定义路径
